PROVENFOLIO OWNER'S MANUAL

Provenfolio 完全使用ガイド

あなたの GitHub のリポジトリを AI が分析して、証拠付きのポートフォリオサイトを自動で作り、育て続けるシステムです。このガイドは、エンジニアの知識がなくても全機能を使いこなせるように書かれています。

SECTION 01

全体のしくみ — 3つの約束

Provenfolio は「盛らないポートフォリオ」を作るための仕組みです。裏側では次の流れが動いています。

このシステムには、信頼性を守るための3つの約束が組み込まれています。

  1. 数字はぜんぶ実測。 コミット数・開発日数・言語比率などサイトに出る数値は、すべてプログラムが git 履歴から自動計測したものだけ。AI が数字を「盛る」ことは構造的にできません。
  2. 主張には証拠リンク。 「起動を大幅に短縮」のようなアピールには、必ず根拠(コミット・ファイル・PR)へのリンクが付きます。根拠がない主張はサイトに載りません。
  3. 文章の最終所有者はあなた。 AI が下書きを書きますが、あなたが編集した文章は再分析でも勝手に上書きされません(編集済みを自動検知します)。
ヒント

data/ フォルダ = あなたの財産。プロフィール・作品データ・スクショが全部ここに入っています。他はぜんぶ「エンジン(道具)」で、data/ さえあればサイトはいつでも作り直せます。

SECTION 03

作品を追加する — いちばんよく使う操作

GitHub のリポジトリの URL を渡すだけで、分析 → 文章の下書き → 翻訳まで自動で進みます。

CLAUDE にこう言う 「/analyze https://github.com/(あなたのアカウント)/(リポジトリ名) 」

「この作品をポートフォリオに追加して: (URL)」でも通じます。GitHub にない作品は「手動で作品を追加したい」と言えば OK。

「あなたの出番」で聞かれること

機械には分からない、本人しか知らない情報だけを聞かれます。Studio の「インテーク」タブから後で書き足すこともできます。

  • カテゴリ — プロダクト / 受託 / OSS / 趣味 / 学習 のどれ?
  • 役割 — ひとりで作った? チームでの担当は?
  • 動機 — なぜ作った? 何を解決したかった?
  • デモリンク — 動くものを見せられる URL(ストア・動画も可)
  • 数字で語れる成果 — ユーザー数・DL数・速度改善など(あれば)
ヒント

追加した作品は公開するまでネットには出ません。何個でも気軽に追加して、Studio で仕上がりを見てから公開しましょう。

SECTION 04

Studio(管理画面)の使い方

Studio はあなた専用のコックピット。このパソコンの中だけで動き、ネットには公開されません。

SECTION 05

文章を直す・翻訳を追いつかせる

AI の下書きは「たたき台」。あなたの言葉に直してこそポートフォリオです。方法は2つ:

  • Studio の「プロース」タブ — 自分でテキストボックスを直接編集(直したその場で保存)
  • Claude に頼む — 会話で直したいときはこちら
CLAUDE にこう言う 「/edit (作品名) — 概要文をもっと簡潔に、成果を先頭に持ってきて」

元の文章を直したら翻訳版も自動で追従させてくれます。「翻訳だけやり直して」もOK。

たいせつ

あなたが編集した文章は保護されます。再分析しても上書きされません。AI に書き直させたい場合だけ、その旨を明示的に伝えてください。

SECTION 06

スクリーンショットを載せる

  1. Studio で作品を選び、「メディア」タブを開く対応形式: png / jpg / webp / gif(1枚 10MB まで)
  2. 「ファイルを選択」→「アップロード」複数まとめて選択できます
  3. alt テキスト(画像の説明)を各言語で入力目の不自由な閲覧者や検索エンジンのための説明文。「メイン画面。検索結果に引用元ページが表示されている」のように

画像は data/assets/screenshots/作品名/ に保存され、その作品のページに自動で並びます。アーキテクチャ図(構成図)も同じ手順で載せられます。

SECTION 07

デザインを変える — 7つのテーマと訪問者スイッチャー

サイトの見た目は「テーマ」として丸ごと差し替えできます。色だけでなくレイアウトも書体もすべて変わります。7種類を同梱済み。

Midnight ●
既定midnight

ダークで静かな夜のトーン。3D の星座ヒーローが動く。迷ったらこれ。

Editorial Swiss ●
穏editorial-swiss

白い紙に黒い活字と赤い罫線。年次報告書のような知的な佇まい。

Quiet Luxury ●
穏quiet-luxury

生成りの紙にセリフ書体とクレイ色。上質な作品集のような温かさ。

Neo-Brutalist ●
中neo-brutalist

太い罫線と絶叫オレンジ。スタンプが押された貨物伝票のような迫力。

Data Forensic ●
中data-forensic

チャコールに蛍光グリーンの端末風。全データに「証拠」の空気をまとう。

Kinetic Type ●
狂kinetic-type

画面を突き破る特大タイポグラフィがスクロールで動く。見世物系。

Acid Lab ●
狂acid-lab

紫黒にクロームとアシッドイエロー。Y2K 実験室。攻めたいときに。

① 既定デザインを変える(Studio から)

  1. Studio 左下の「サイトのデザイン」を押す
  2. 「既定のデザイン」で既定テーマを選ぶサイトのメイン URL で最初に表示されるデザイン。検索エンジンが見るのもこれ
  3. 「訪問者スイッチャー」で訪問者に見せたいテーマにチェック
  4. 「デザインを保存」→ 次の公開(§8)で反映

② 訪問者も切り替えられる(サイトの右下)

公開サイトの右下に「DESIGN」ボタンが常駐していて、訪問者自身がその場でデザインを丸ごと切り替えられます。これ自体が技術力のデモになる、Provenfolio の名物機能です。

③ 新しいテーマが欲しくなったら

CLAUDE にこう言う 「/design propose — 和モダンな方向で3案見せて」

実在の有名サイトをリサーチした複数のデザイン案がブラウザに並び、選んだ案が本物のテーマとして実装されます。「/design brief」で好みを登録しておくと、以降の提案があなた好みに寄ります。

SECTION 08

プレビューと公開

公開前に必ずプレビュー

CLAUDE にこう言う 「/update-site」

データ検査 → サイト生成 → ブラウザで表示までやってくれます。自分で見るだけなら npm run dev → localhost:4321 でも。

公開する

CLAUDE にこう言う 「/publish」

完成度チェック → ビルド → 「本当に公開しますか?」の確認 → 公開、まで進みます。確認なしに勝手に公開されることはありません。

環境によっては、Claude がセキュリティ制限で最後のデプロイだけ実行できないことがあります。その場合は Provenfolio のフォルダで、この1行を自分のターミナルに:

TERMINAL — 公開コマンド
npx wrangler pages deploy site/dist --project-name (あなたのプロジェクト名) --branch main

数秒であなたの公開 URL に反映されます。公開は何度でもやり直せます(毎回まるごと置き換え)。

SECTION 09

週次の自動更新(設定すると完全放置できる)

一度設定すると、毎週金曜 21:00 UTC(日本時間では土曜の朝6時)に「開発の数字を測り直し → サイト再生成 → 公開」まで全自動で回ります(時刻は refresh.yml で変更できます)。あなたが普段どおり開発するだけで、ポートフォリオの数字が勝手に最新になります。

初回設定(10分・一度だけ)

「鍵(APIトークン)」を GitHub に預ける作業なので、ここだけはあなた自身の手で行います(Claude は認証情報を扱えない決まりです)。Cloudflare Pages で公開している前提の手順です。

  1. Cloudflare のアカウント ID をコピーdash.cloudflare.com にログイン → URL の dash.cloudflare.com/〈この部分〉 がアカウントID
  2. API トークンを作る右上プロフィール → My Profile → API Tokens → Create Token → Create Custom Token。Permissions に「Account / Cloudflare Pages / Edit」を1行追加 → Create。表示されたトークンをその場でコピー(再表示不可!)
  3. GitHub に Secrets を2つ登録あなたのインスタンスリポジトリ(GitHub)→ Settings → Secrets and variables → Actions → Secrets タブ → New repository secret で CLOUDFLARE_API_TOKEN(手順2の値)と CLOUDFLARE_ACCOUNT_ID(手順1の値)
  4. Variable を1つ登録同じ画面の Variables タブ → New repository variable で CF_PAGES_PROJECT = あなたの Pages プロジェクト名
  5. 動作確認リポジトリの Actions タブ → 「refresh」→ Run workflow。緑のチェックが付けば設定完了
ヒント

非公開リポジトリを分析対象にする場合だけ、追加で Secret GH_TOKEN(GitHub の Personal Access Token、repo 読み取り権限)が必要です。公開リポだけなら不要。

SECTION 10

エンジンのアップデートを受け取る

Provenfolio 本体(エンジン)は配布リポジトリで開発が続きます。新機能や新テーマが出たら取り込めます — あなたの data/(作品・文章・写真)は構造的に絶対に壊れない設計です。

① Studio のボタンで(いちばん簡単)

  1. Studio 左下の「エンジン更新」を押す
  2. 「更新を確認」→ 新着があれば内容の一覧が出る
  3. 「今すぐ更新」→ 取得・統合・再ビルドまで自動で流れるログが画面に流れます。統合(merge)に失敗しても自動で元に戻ります
  4. 完了したら Studio を再起動新しいエンジンで動き始めます

② Claude に頼む

CLAUDE にこう言う 「エンジンの更新を取り込んで」

取得 → 統合 → 依存更新 → データ移行 → 検査 → ビルド確認まで全部やります。更新の衝突など、ボタンで失敗したときの解決もこちらで。

SECTION 12

困ったとき Q&A

Q. テーマを変えたのにサイトが変わらない

デザイン変更は次の生成・公開時に反映されます。「/update-site して」でプレビュー確認 → 「/publish」で本番へ。プレビューサーバーを開きっぱなしだった場合は再起動が必要です(「dev サーバーを再起動して」)。

Q. 「ポートが使用中」と言われて起動できない

CLAUDE にこう言う「ポート 4600 を塞いでるプロセスを止めて、Studio を起動し直して」

Q. ビルドやコマンドが赤いエラーで止まった

CLAUDE にこう言う「npm run validate を実行して、エラーを直して」

エラー文をそのまま貼り付けるのが最速です。

Q. エンジン更新が途中で止まった

Studio のログに理由が出ています。統合(merge)に失敗した場合は自動で元の状態に戻ります。それ以降の工程(依存関係の更新やビルド)で止まった場合は、更新は取り込まれたものの検査が済んでいない状態です。どちらでも、「エンジンの更新を取り込んで。さっき Studio で失敗した」と Claude に頼めば、原因ごと解決してくれます。

Q. サイト右下の DESIGN ボタンが消えた

訪問者に見せるデザインが実質1つ(既定テーマのみ)になると自動で隠れます(仕様)。Studio の「サイトのデザイン」で、既定テーマ以外にも1つ以上チェックを入れてください。

Q. 公開をやめたい/失敗した公開を戻したい

公開は毎回まるごと置き換えなので、直したものを再公開すれば上書きされます。サイト自体を消したい場合は Cloudflare ダッシュボード → Workers & Pages → あなたのプロジェクト → Settings → Delete。

Q. なにをどこまで Claude に任せていいの?

全部任せて大丈夫です。ただし次の3つは必ずあなたの確認や操作を挟む設計になっています: ① ネットへの公開(/publish)、② リポジトリへの反映(push)、③ パスワードや API トークンの入力(これは Claude は代行できません)。

用語ミニ辞典

リポジトリ
プログラムの保管箱。変更の履歴ごと保存されている。GitHub はその置き場サービス

「ここまでやった」というセーブ記録。Provenfolio はこれを数えて開発実績を測る

データからサイト(HTML の束)を焼き上げること。手元で何度でも無料でできる
デプロイ / 公開
焼き上げたサイトをインターネットに置くこと。ここで初めて世界から見える
プレビュー
公開前に自分だけで見ること。localhost(自分のPC内)で動く
テーマ
サイトの見た目一式(配色・書体・レイアウト)。着せ替えのように交換できる
インテーク
作品について本人しか知らない情報(役割・動機など)の受付フォーム
ケーススタディ
作品1つぶんの紹介記事。課題 → 解決 → 成果の構成で自動下書きされる
完成度スコア
作品ページの仕上がり度(100点満点)。足りない項目も教えてくれる
API トークン
サービスを操作するための合鍵。パスワードと同じ扱いで、人にもAIにも渡さない(GitHub Secrets が正しい保管場所)
Cloudflare Pages
サイトの無料ホスティング(置き場)サービス。既定の公開先
エンジン / インスタンス
